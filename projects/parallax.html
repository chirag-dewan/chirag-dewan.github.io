PARALLAX

Privacy-preserving behavioral threat detection for AI platforms

Thesis

See the pattern, not the content. Detect AI platform abuse — prompt injection, model extraction, data exfiltration, automated exploitation — through behavioral metadata analysis without ever inspecting user content. Privacy by architecture, not by policy.

Architecture

Three-tier detection pipeline operating on API event metadata:

JSONL events
  → DetectionPipeline.load_traffic()
  → Pydantic APIEvent parsing
  → Per-account AccountProfile construction
  → PopulationBaseline (cross-account stats)
  → 14 independent detectors (8 Tier 1, 6 Tier 2)
  → Weighted composite scoring
  → ThreatAssessment

Tier 1 — Account-Level Signals (72% weight)

Eight detectors analyzing individual account behavior against expected patterns:

  • T1-001 Request velocity — burst detection and sustained rate analysis
  • T1-002 Temporal concentration — entropy-based session clustering
  • T1-003 Token ratio — input/output asymmetry (extraction signature)
  • T1-004 Error rate exploitation — systematic probing patterns
  • T1-005 Model targeting — enumeration and capability mapping
  • T1-006 Context window exploitation — boundary-pushing behavior
  • T1-007 Endpoint diversity — API surface coverage anomalies
  • T1-008 Session pattern — behavioral consistency over time

Tier 2 — Population-Level Signals (28% weight)

Six detectors comparing account behavior against population baselines using z-score analysis:

  • T2-001 Statistical outlier — multi-dimensional deviation scoring
  • T2-002 Usage pattern anomaly — behavioral clustering distance
  • T2-003 Rate deviation — population-relative velocity analysis
  • T2-004 Resource consumption — compute/token budget anomalies
  • T2-005 Behavioral consistency — profile stability over time
  • T2-006 Cross-account correlation — coordinated behavior detection

Composite Scoring

composite = Σ(score × weight × confidence) / Σ(weight)

Threat levels:
  NONE     < 0.25
  LOW      0.25 – 0.49
  MEDIUM   0.50 – 0.69
  HIGH     0.70 – 0.84
  CRITICAL ≥ 0.85

Tech Stack

Python 3.11+ · Pydantic models · Statistical anomaly detection · Flask API · pytest (92 tests, 94% coverage)

Validation

Evaluated against LANL Unified Host and Network Dataset (real authentication logs from Los Alamos National Laboratory). Adversarial test suite covers four evasion scenarios: blended behavior, low-and-slow extraction, single-signal ablation, and threshold sensitivity.

Status

Active development. 14 detectors implemented and tested. Pipeline operational. Working toward windowed scoring and signal correlation for next phase.
