KESTREL

Catching attacks on cloud infrastructure by learning each account’s normal and flagging the weird.

The problem

PARALLAX watches what users do to an AI platform. KESTREL watches the machines underneath. Picture someone quietly hijacking your GPU cluster to mine crypto, a rogue training job burning compute for two days on a stolen account, or a recon tool listing every permission across thirteen cloud regions at 2 a.m. None of that shows up in app behaviour — it shows up in the cloud’s own logs.

The idea

Learn what normal looks like for each account, then flag anything that drifts more than a couple of standard deviations from its own habits. No fixed thresholds, no training data — every account is judged against itself.

What I built

Three detectors — GPU hijacking, rogue training, and cloud-API abuse — that tag each finding with MITRE ATLAS techniques and export straight to Sigma rules a SIEM can ingest. It worked beautifully on my own synthetic data, which should always make you suspicious.

The reality check

I ran it against 34,427 real CloudTrail events full of genuine attacks — IAM enumeration, privilege escalation, recon scanning. Precision: a perfect 1.0. Recall: 0.0005. It caught one thing out of 2,167 attack events. Flawless aim, pointed at the wrong target.

The diagnosis

Classic mismatch. My detectors were hunting for GPU telemetry — utilisation, power draw, training durations. But CloudTrail is an API audit log: 99.98% of the events were API calls, and exactly zero were GPU metrics. I’d built the right instrument for data that simply wasn’t there. (The baseline engine even spotted the attacker scanning at 11 p.m. — nothing was listening to that signal.)

The fix

So I built a detector for the data I actually had: API bursts, fanning out across regions, off-hours spikes, and walls of “access denied.” Recall went from 0.0005 to 0.766 — a 1,532× improvement — while precision stayed at 1.0, zero false positives. 220 findings, and every account v1 missed got caught.

The fix wasn’t fancier machine learning. It was meeting the data where it lives: diagnose the mismatch, build the detector that fits. Subtraction before addition, again.
