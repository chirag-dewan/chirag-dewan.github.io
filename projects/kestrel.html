KESTREL

Cloud-native AI workload anomaly detection

Thesis

Baseline normal AI workload behavior, detect deviations. Rolling per-entity z-score analysis catches GPU hijacking, rogue training jobs, and unauthorized model access before they escalate.

Architecture

Synthetic event generator
  → SQLite storage
  → Rolling z-score baselining engine
  → Detection modules (GPU hijack, rogue training)
  → CLI surface + Flask dashboard + Sigma rule export

CLI

kestrel generate   # Synthetic event generation
kestrel scan       # Run detection pass
kestrel serve      # Launch Flask dashboard
kestrel findings   # Export findings
kestrel version    # Version info

Relationship to PARALLAX

PARALLAX operates at the application layer — API request patterns, user behavior, content-free metadata. KESTREL operates at the infrastructure layer — GPU utilization, training job profiles, compute resource anomalies. Together they provide full-stack AI platform threat detection.

Status

Architecture locked. Twelve specification files produced for implementation. PARALLAX's infrastructure-layer companion.
