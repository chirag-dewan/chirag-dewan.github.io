KESTREL

Cloud-native AI workload anomaly detection

Thesis

Baseline normal AI workload behavior, detect deviations. Rolling per-entity z-score analysis catches GPU hijacking, rogue training jobs, and unauthorized model access before they escalate.

Architecture

CloudTrail adapter / Synthetic generator
  → SQLite storage
  → Rolling z-score baselining engine (Welford's algorithm)
  → Detection modules:
      GPUHijackDetector    — GPU utilization, anomalous regions, off-hours
      RogueTrainingDetector — instance types, duration z-scores, data transfer
      CloudAPIAbuseDetector — API burst, region fan-out, error rates, off-hours
  → Findings (MITRE ATLAS tagged) → Sigma rule export → Flask dashboard

Evaluation

Evaluated against 34,427 real CloudTrail events from Splunk Attack Data (51 datasets, 77 entities, 16 AWS regions).

v1: Precision 1.0 | Recall 0.0005 | 1 finding
    → Impedance mismatch: GPU detectors vs API audit logs

v2: Precision 1.0 | Recall 0.766  | 220 findings
    → CloudAPIAbuseDetector processes 99.98% of CloudTrail data
    → 1,532x recall improvement, zero false positives

Key detections: daftpunk (off-hours + 100% error rate → HIGH), cloudmapper (burst + 13-region fan-out → HIGH), bhavin_cli (24 findings), okta_ro_role (292 calls, 12 regions → HIGH).

Relationship to PARALLAX

PARALLAX operates at the application layer — API request patterns, user behavior, content-free metadata. KESTREL operates at the infrastructure layer — GPU utilization, training job profiles, API abuse patterns. Together they provide full-stack AI platform threat detection.

Status

v2 complete. Three detectors, 98% test coverage, real-data evaluation. PARALLAX's infrastructure-layer companion.
